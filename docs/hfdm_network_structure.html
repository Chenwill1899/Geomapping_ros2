H-FDM H25 Learned Rollout Network

Geomapping MPPI currently uses a TorchScript export from high_level_fdm. The learned model predicts relative future poses, applied twists, and stepwise risks for each MPPI candidate sequence; the controller then adds goal/path/progress and residual regularization costs around that learned rollout.

Figure 1. Runtime Dataflow

Geomapping Inputs

  • Robot state: [x,y,yaw,vx,vy,wz]
  • Local costmap: reward_cost
  • Frontend path and RViz goal

History Buffer

Last 10 relative poses, measured velocities, and previous commands are packed into [B,10,10].

Candidate Controls

MPPI samples 896 action sequences, each shaped [25,3] for vx,vy,wz.

Robot-Centered Patch

The costmap is sampled into a normalized [B,1,64,64] local patch. Missing map data becomes a zero or unknown-cost patch depending on runtime state.

HighLevelFDM

  • History GRU -> context embedding
  • Small CNN -> map embedding
  • Action MLP + path map samples
  • Decoder GRU -> future hidden states

Learned Consequences

  • Relative pose: [B,25,4]
  • Risk logits: [B,25,6]
  • Applied twist: [B,25,3]

MPPI Objective

Learned rollout replaces handwritten kinematics. The controller still adds goal, yaw, path, progress, smoothness, acceleration, jerk, lateral, yaw-rate, and learned-risk costs.

Runtime source: src/mppi_controller/mppi_controller/controllers/mppi_omni_high_level_fdm_torch.py and src/mppi_controller/mppi_controller/core/high_level_fdm_runtime.py.

Figure 2. Layer-Level Architecture

Inputs

History Tensor

[B,10,10]

Relative pose x,y,sin(yaw),cos(yaw), measured velocity vx,vy,wz, and previous command vx,vy,wz.

Local Map

[B,1,64,64]

Single-channel normalized reward_cost, max cost 100.0, 0.10 m per cell.

Action Sequence

[B,25,3]

Candidate controls over 2.5 s: vx, vy, wz, with dt=0.1.

Encoders

HistoryEncoder

GRU(10 -> 128) 1 layer

Returns the final hidden state z_hist.

SmallMapEncoder

Conv 1 -> 32, k5, s2 Conv 32 -> 64, k3, s2 Conv 64 -> 128, k3, s2 AvgPool + MLP 128 -> 128

Produces z_map, a global terrain context vector.

ActionEncoder

MLP 3 -> 32 -> 32

Each future control is embedded independently as a_emb.

Path Conditioning

Nominal Twist Integration

[B,25,3]

Raw actions are integrated with dt=0.1 to form a nominal relative path before residual correction.

nominal_xytheta = integrate_twist(actions)

Map Samples Along Path

offsets: 0, +2, -2 cells path_feat = [B,25,3]

Samples the local map at the centerline and two lateral offsets around the nominal trajectory.

Fusion + Recurrent Decoder

Feature Concatenation

32 + 128 + 128 + 3 = 291

For each future step: action embedding, broadcast history embedding, broadcast map embedding, and path-conditioned map samples.

sequence model

Decoder GRU

GRU(291 -> 128) 2 layers dropout 0.10

Produces one hidden state per future step: dec_out [B,25,128].

Prediction Heads

Delta Twist Head

MLP 128 -> 128 -> 64 -> 3

Predicts bounded residual twist, scaled by learned delta_scale=[0.5,0.5,0.8].

applied_twist = actions + tanh(head(dec_out)) * delta_scale

Risk Head

MLP 128 -> 128 -> 64 -> 6

Outputs logits for collision, high_cost, stuck, tracking_failure, untraversable, and any.

Outputs + MPPI Use

Pose Rollout

pose [B,25,4]

Applied twist is integrated, then yaw is represented as sin/cos. Geomapping converts this relative pose to world states.

Learned Risk Cost

risk_any index

Uses the any risk channel, threshold 0.6, power 2.0, and weight 4.0 in the current H-FDM profile.

cost = 4.0 * sum(max(risk_any - 0.6, 0)^2)

Retained Objective Terms

Goal/yaw, path tracking, path progress, goal progress, heading-to-goal, control, smoothness, acceleration, jerk, lateral, and yaw-rate terms remain outside the learned model.
